Where we are · 8-slice build

1 2 3 4 5 6 7 8 9+beyond

Phase 1 is complete — all 8 slices merged. The determinism engine: reversible engine (ad4e9c0), filesystem adapter (c926f97), irreversible lane (a13018b), isolation/MVCC (0fab344), replay (9ff69a6), policy engine (c2318d5), speculative dry-run (6ee8980), and the MCP gateway (9d41df8) — Pherix is now both a Python library and an MCP server driving the same core. The gateway landed with core/ essentially untouched (one nullable client_id audit column + the absorbed StateDiffable sub-protocol), so the front-end-agnostic discipline held across all eight slices. The real-agent dogfood suite has since landed — a shared harness + DevOps / audit / coding-sandbox / live-MCP streams — and on day one the audit dogfood surfaced a genuine Slice-4 isolation bug all the deterministic tests missed; it's now fixed (merged), local agents + OpenClaw are wired in (the harness drives a local model via --local; an air-gapped capstone proves model- and deployment-blindness), and the accuracy pass landed — the runnable dogfoods are now genuinely autonomous (de-railed prompts, un-rigged outcomes) with a capture.py evidence harness. 396 tests green. Phase 2 now — the guardrails MVP: run the dogfoods for real (the first-user signal), land a design partner, stand up the audit pillar, toward a YC application. See phases.html for the three-act roadmap and dogfood.html → Start here to run the agents. Beyond: the trilogy of determinism, memory, audit — see future.html.
