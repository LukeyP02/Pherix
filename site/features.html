Features

What Pherix gives you.

The guardrails Pherix puts around your agent's tool calls. Everything below is built and tested today.

These ship as a library plus a read-only inspector — not a SaaS dashboard or hosted service. You pip install it, wrap your tool-call layer, and self-host. Pre-release 0.0.0.

Containment

Reversible effects — database and file writes — roll back through the backend's own savepoints and copy-on-write. The backend does the undo, exactly. What can't be undone runs a registered opposite instead: a refund for a charge, a deprovision for a provision.

Oversight

An irreversible effect with no safe undo is held at commit, not fired — until a human (or a higher-trust policy) approves it with approve_irreversible.

Audit

Every tool call is recorded as it happens — the journal is the audit log, no separate logging to trust. Read it back in the inspector, or re-run it to confirm it still does the same thing.

Policy

Allow and deny rules, plus spend and count caps, checked at commit and again when an effect is staged — a bad call fails early, not at the end of a run.

Isolation

When two agents run at once, Pherix tracks what each reads and writes and catches conflicts before they corrupt shared state.

Dry-run

Run against a snapshot, then throw it away — see what a run would do before it touches anything real.

Governed memory

The same guardrails pointed at the agent's memory — what it remembers stays contained and on the record too.
