Python API

Everything you import from pherix. Click an entry to expand its signature and notes.

Interception

tool(resource, *, reversible=True, injects_handle=True, compensator=None)

Decorator that marks a function as a side-effecting tool. resource names which adapter handles it. Set reversible=False for effects that cannot be snapshotted (they take the staged/gated lane). compensator names a registered inverse tool to use on rollback. injects_handle=False if the tool does not take a resource handle as its first argument.

@tool(resource="sql")
def insert_user(conn, name, role): ...
agent_txn(adapters, *, policy=None, audit=None, client_id=None) → context manager

Open a transaction. adapters maps each resource name to a ResourceAdapter. Effects journalled inside the block commit on clean exit. Yields a TxnContext for mid-sequence control.

with agent_txn({"sql": SQLiteAdapter(conn)}, policy=p, audit=j) as txn:
    ...
TxnContext — txn.txn_id · txn.rollback() · txn.commit() · txn.approve_irreversible(effect_id)

The object yielded by agent_txn. rollback() folds the journal backward (restore/compensate each effect). commit() is implicit on clean block exit but can be called explicitly. approve_irreversible(effect_id) releases a gated irreversible effect that has no compensator.

Policy

Policy(rules=[...]) · Allow(...) · Deny(...) · Cap.sum(...) / Cap.count(...)

A policy is a list of rules evaluated at stage-time and again at commit-time. Allow/Deny gate by tool and arguments; Cap.sum bounds the total of a numeric arg across the transaction; Cap.count bounds how many times a tool fires.

Policy(rules=[
    Deny(tool="charge_card", when=lambda a: a["cents"] > 50_000),
    Cap.sum(tool="charge_card", arg="cents", limit=100_000),
])
PolicyViolation · GateBlocked · PolicyVerdict

PolicyViolation is raised when a rule denies an effect. GateBlocked is raised at commit when an irreversible effect is gated and unapproved. PolicyVerdict is the per-rule decision recorded in the journal.

Isolation

Serialize · Retry · Abort — concurrent-transaction conflict policies

Pass an isolation policy to agent_txn to choose what happens when two transactions touch the same keys. Abort fails the conflicting commit; Retry re-runs it; Serialize waits for the in-flight writer. Conflicts are detected by diffing read/write keys against the journal.

Dry-run

dry_run(adapters, *, policy=None) → DryRunResult

Fold the journal forward against a snapshot, then discard it — see what a sequence would do (rows added, files written, what would fire) without committing anything.

with dry_run(adapters) as ctx:
    plan_changes()
# ctx carries the would-be plan; the world is untouched

Audit

AuditJournal(path) · AuditJournal.in_memory() · get_transaction · get_effects · get_verdicts

The durable transcript. Pass a path for persistence (replay needs it) or in_memory() for ephemeral runs. Read it back per transaction.

Sync (control-plane ship side) pherix[sync]

JournalShipper(journal, agent_id, transport, *, key=None, redactor=None, host=None)

Forward-reads the journal past a durable cursor, redacts PII, encrypts the payload under your key, and ships batches to a control plane — never blocking the agent. pump() drains once; start()/stop() run it on a daemon thread.

PayloadCipher(key) · generate_key() · Redactor(keys=None)

AES-256-GCM payload encryption (BYOK) and a field-name PII redactor. With a key, shipped effect args/result are ciphertext only you can read; metadata (tool/resource/status/ts) stays cleartext.
