Guardrails for AI agents

Let your agent run.
Undo the mistakes.

A self-hosted library that wraps your agent's tool calls. Every action is reversible, gated, and audited — so a bad run is one rollback away, not an incident.

Watch the demo $pip install -e . Get started

The problem

An agent calls a tool and the effect just happens — a wiped table, a wrong payment, a clobbered file. No undo, no approval, no record. Pherix makes that contained, not catastrophic.

What you get

01

Undo the reversible

DB and file writes roll back through the backend's own savepoints — exact, not best-effort.

02

Gate the irreversible

A charge or email can't be un-sent — so it waits for a human to approve before it fires.

03

Audit everything

Every tool call recorded — who did what, when, allowed or denied. Always on.

Where it earns its place

Regulated data

Agents on customer & financial records — legal-hold rows can't be deleted, regulated data can't leave the boundary, every change audited.

Payments & money movement

Money moves only on human sign-off; a refund is its registered undo; the trail clears fraud and finance review.

Production systems

Prod write access without the fear — a bad change rolls back to the exact prior state, with the gate and audit change-management needs.

How it fits

A library you pip install and wrap your agent's tool layer with — your loop and model unchanged. It doesn't run agents; it makes their actions behave like a database transaction. Self-hosted, pre-release.
