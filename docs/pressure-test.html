Phase 3 — Adversarial probes (executed today)

PROBE 1 · KNOWN-BROKEN CONFIRMED

A reader tool that returns its cursor crashes the whole transaction

The latent bug CLAUDE.md flags is reachable through the public API, not just theoretical. A perfectly natural @tool — return execute_isolated(conn, "SELECT ...") — kills the txn at journal-write time, because strict_json_default refuses to serialise a Cursor. The existing suite hides it because every test .fetchall()s before the result reaches audit.

$ python probe_cursor.py
RESULT: BUG REPRODUCED — TypeError: Pherix cannot journal 'Cursor'
  raised from: effects.py:64, in strict_json_default

Fix: materialise readers via .fetchall() in execute_isolated before return; ship the probe as a permanent regression test. ~10 lines.

PROBE 2 · DEAD TEST CONFIRMED

The Postgres moat-claim holds — but its test silently never runs

test_laws_differential.py:144 imports PostgresAdapter from the wrong module (adapters.sql instead of adapters.postgres), hits ImportError, and skips with the now-false reason "PostgresAdapter not implemented yet." A live PG is reachable on this box. I fixed the import in a probe and ran the cross-backend rollback law directly:

Fix: correct the import, drop the second pytest.skip stub, fill in the differential body. The capability is real; only the CI proof is missing.
