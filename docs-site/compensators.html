Compensators

A compensator is a semantic inverse — an opposite action that undoes an effect that can't be snapshotted. Charge → refund, invite → revoke, provision → delete. Where an adapter reverts state, a compensator runs a counter-action.

When a compensator is used

An irreversible effect (a tool with reversible=False) is staged, not run live, and fires at commit:

  • With a compensator registered — it auto-commits, and the compensator runs on rollback to undo it.
  • Without one — commit gates: it blocks until an explicit txn.approve_irreversible(effect_id).

A rollback before commit needs no compensator at all — the staged effect simply never fired.

Example: charge that auto-refunds on rollback

from pherix.compensators import register_charge_refund

# client must expose: charge(idempotency_key, amount_cents, currency) and
# refund(idempotency_key). The pair reverses by the idempotency key.
charge, refund = register_charge_refund(stripe_like_client)

with agent_txn(adapters) as txn:
    charge(idempotency_key="ord_42", amount_cents=4200)   # staged
    do_other_work()
    if failed:
        txn.rollback()    # charge had not fired yet → nothing to refund
# clean exit → charge fires at commit; a LATER rollback would run refund("ord_42")

Writing your own

Declare the action with reversible=False and name its inverse via compensator=. The compensator is an ordinary tool; the runtime fires it with the action's argument set on rollback.

@tool(resource="billing", reversible=False, injects_handle=False)
def refund(idempotency_key, amount_cents, currency="usd"):
    return client.refund(idempotency_key)

@tool(resource="billing", reversible=False, injects_handle=False, compensator="refund")
def charge(idempotency_key, amount_cents, currency="usd"):
    return client.charge(idempotency_key, amount_cents, currency)
A good compensator is a left-inverse: compensator ∘ action ≈ identity. Key it off an idempotency key (as real payment APIs do), so a retried compensation is a no-op and unambiguously targets the action it inverts — even though the compensator only ever sees the action's arguments, never its return value.
