Features

What Pherix gives you.

The governance primitives Pherix puts around your agent's tool calls — grouped by the question each one answers. Everything below is built and tested today.

These are primitives delivered as a library plus a read-only audit console — not a SaaS dashboard or hosted control-plane. You pip install it, wrap your tool-call layer, and self-host. Pre-release 0.0.0.

The seven capabilities

Control

policy

Allow/deny rules plus caps — sum and count limits — and world-state-aware rules that read live external state before deciding.

Enforced at commit, and again at stage-time so a bad call fails fast rather than at the end of a run.

Containment

adapters + compensators

Reversible effects roll back via real backend semantics — database savepoints, filesystem copy-on-write. The backend does the undo, exactly.

Irreversible effects use compensators: vetted semantic inverses that run an opposite action when an exact rollback is impossible.

Oversight

the gate

An un-compensable irreversible effect is staged, not fired. It blocks at commit until explicitly approved via approve_irreversible.

The human — or a higher-trust policy — is the only thing that lets it through.

Audit

the journal

Every side-effecting tool call is an append-only journal entry — the journal is the audit log, no separate logging to trust.

Read it back with the read-only inspector console (python -m pherix.inspector), or replay it deterministically.

Isolation

MVCC

MVCC-style isolation between concurrent agents. Each transaction tracks its read and write sets; conflicting effects are detected before they corrupt shared state.

Dry-run

speculative

Fold the journal forward against a snapshot, then discard it. Speculative execution with no real effects — see what a run would do before letting it touch anything.

Governed memory

adapter + policy

An adapter and a policy pointed at the agent's memory — the same primitives applied to what the agent remembers, so memory writes are contained and on the record too.
