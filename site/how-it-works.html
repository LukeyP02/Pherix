// how it works

One wrapper. Every effect becomes undoable.

Pherix sits at your agent's tool-call layer and turns each side effect into a transaction — so a whole run is all-or-nothing, dangerous calls need approval, and everything is on the record. Your agent loop and model stay exactly as they are.

// the shape of a run

1 · intercept
A tool call is caught and journalled — never fired blindly.
→
2 · route
Reversible? Run it live, snapshotted. Irreversible? Stage it behind policy.
→
3 · settle
Clean exit commits the run. Any failure rolls the whole thing back.
# your agent loop, unchanged — just wrapped with agent_txn(adapters, policy) as txn: agent.run(task) # tool calls journalled + snapshotted live # commit on clean exit — any exception rolls the whole run back

// two kinds of effect, two ways to undo

Reversible

snapshot, then restore

Database writes, file writes — anything the backend can roll back. Pherix takes a snapshot before the effect runs and restores it on failure. The backend does the undo, so it's exact.

e.g. SQL savepoint, copy-on-write files → roll back byte-identical.

Irreversible

stage, then gate or compensate

Charges, emails, outbound API calls — things no snapshot can undo. They're held until commit, then either gate for human approval, or auto-undo with a registered compensator.

e.g. charge → refund, invite → revoke, or block for sign-off.

// the two things that come for free

Policy decides

One clean slate of rules — caps, allow/deny, live-state checks, the human gate. Checked when an effect is staged and again at commit, so nothing slips through on stale state.

The journal records

Every effect — attempted, allowed, denied, undone — is an entry in an append-only log. The audit trail isn't a feature you turn on; it's just the journal, read back.

Free and open source.

The runtime is an open-source library. Install it, wrap your tools, and run it entirely yourself — there's nothing to phone home to.

  • Both lanes, the adapters, policy — the whole engine
  • Python & TypeScript SDKs
  • No account, no cloud, fully offline
  • Keep your existing agent loop and model
