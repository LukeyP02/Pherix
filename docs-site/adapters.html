Adapters

An adapter is a triple (snapshot, apply, restore) over a class of resource — the seam that makes a backend transactable. The backend does the undo; Pherix orchestrates the fold.

The catalog

The kernel has zero third-party dependencies. Each backend driver is an optional extra, lazy-imported only when you construct that adapter — so import pherix works with nothing installed.

Reversible vs. irreversible. The first group can be snapshotted, so a rollback restores exact state. HTTP/REST cannot — they take the staged/gated lane and rely on compensators for semantic undo. Pherix is honest about what it cannot revert rather than pretending.

Example: bind two resources

from pherix import SQLiteAdapter, S3Adapter, agent_txn

adapters = {
    "sql": SQLiteAdapter(conn),
    "s3":  S3Adapter(boto3_client, bucket="my-bucket"),
}
with agent_txn(adapters) as txn:
    write_row(...)        # resource="sql"  -> SQLiteAdapter
    put_object(...)       # resource="s3"   -> S3Adapter
    # a rollback restores BOTH: the SQL savepoint and the S3 object before-image

The before-image pattern

Stores without a per-write savepoint (Redis, S3, Mongo, DynamoDB, GCS, Elasticsearch) are made reversible the same way: snapshot captures the prior state of each touched key, and restore rewrites it — re-create if it existed, delete if it did not. A tool declares the keys it touches via args["key"] (single) or args["keys"] (a list).

Writing your own adapter

Implement the protocol — four members — and your resource is transactable:

class MyAdapter:
    name = "myresource"

    def supports_rollback(self) -> bool:
        return True                       # honesty flag; False routes to the staged lane

    def snapshot(self, effect):           # capture the before-state
        ...

    def apply(self, effect, tool_fn):     # execute the effect (inject the handle)
        return tool_fn(self._handle, **effect.args)

    def restore(self, handle):            # revert to the captured before-state
        ...

If your store can also version keys, add read_version/write_version and it joins commit-time isolation too.
