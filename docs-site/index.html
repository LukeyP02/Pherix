Transactional resources for AI agents

Pherix wraps your agent's tool-call layer and gives the external side-effects of those calls database-style guarantees: atomicity, isolation, capability enforcement, and durability. Wrap once; keep your existing agent loop and model provider.

An agent today calls a tool and the effect just happens — irreversibly, immediately, with no isolation from other agents. Pherix sits at the tool-call layer and makes those effects behave like a transaction you can commit or roll back.

with agent_txn(adapters) as txn:
    onboard_user(name="ada", role="engineer")   # journalled, live
    charge_card(account="acct_9", cents=4200)    # staged — fires only at commit
    if something_went_wrong:
        txn.rollback()                          # DB writes undone; the charge never happened

One idea: the versioned effect journal

Every side-effecting tool call becomes an effect — an entry in an append-only, ordered log held by a transaction. Every capability is just a traversal of that log:

  • Commit folds the journal forward, applying each effect.
  • Rollback folds it backward, restoring each one.
  • Dry-run folds forward against a snapshot, then discards it.
  • Audit is just reading the journal — it is the log.

Install

pip install pherix                 # the kernel — zero third-party deps
pip install "pherix[postgres]"      # add a backend driver as an optional extra
pip install "pherix[sync]"          # add the control-plane ship side

TypeScript: npm install pherix.

The four axes

Pherix is exactly the answer to the four questions any system must answer to make an action transactional. Read the full guide →

interception

How do actions reach us?

The Python library, the TypeScript SDK, and the MCP gateway wrap your tool-call layer.

policy

What do we allow?

Allow/deny, caps, and a human gate — evaluated at stage-time and again at commit-time.

adapters

How do we undo what can be undone?

Per-resource snapshot / apply / restore — the backend does the undo.

compensators

How do we undo what can't?

A staged / gated lane plus a catalog of vetted semantic inverses.
