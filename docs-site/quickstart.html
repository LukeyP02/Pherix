Quickstart

Wrap a tool, run it inside a transaction, and watch a rollback undo its database writes — no model or API key required.

1. Install

pip install pherix

The kernel has zero third-party dependencies. SQLite works out of the box; other backends are optional extras.

2. Declare a tool

The @tool decorator marks a function as a side-effecting tool and says which resource it touches. Your agent calls it like any function — it never has to know a transaction is in flight.

from pherix import tool

@tool(resource="sql")
def insert_user(conn, name, role):
    conn.execute("INSERT INTO users (name, role) VALUES (?, ?)", (name, role))
    return name
The first argument (conn) is the resource handle. Pherix injects it from the adapter at call time and hides it from your agent's call site — the agent just calls insert_user(name=…, role=…).

3. Run inside a transaction

Bind each resource to an adapter, then open agent_txn. Effects are journalled as they happen. Leaving the block commits; calling txn.rollback() undoes everything.

import sqlite3
from pherix import AuditJournal, SQLiteAdapter, agent_txn

# autocommit (isolation_level=None) — Pherix owns BEGIN / SAVEPOINT / COMMIT
conn = sqlite3.connect("app.db", isolation_level=None)
conn.execute("CREATE TABLE IF NOT EXISTS users (id INTEGER PRIMARY KEY, name TEXT, role TEXT)")

adapters = {"sql": SQLiteAdapter(conn)}
audit = AuditJournal("journal.db")   # durable transcript of every effect

# Transaction 1 — the agent hits a snag and rolls back
with agent_txn(adapters, audit=audit) as txn:
    insert_user(name="ada", role="engineer")
    insert_user(name="grace", role="enginer")   # typo!
    txn.rollback()                            # both inserts vanish

# Transaction 2 — retry cleanly; leaving the block commits
with agent_txn(adapters, audit=audit) as txn:
    insert_user(name="ada", role="engineer")
    insert_user(name="grace", role="scientist")

print(conn.execute("SELECT name, role FROM users").fetchall())
# -> [('ada', 'engineer'), ('grace', 'scientist')]  — only txn 2 survived

4. Read the journal

The audit journal recorded both stories — the rolled-back attempt and the committed one. It is the same log the engine folded; reading it is free.

for effect in audit.get_effects(txn.txn_id):
    print(effect["tool"], effect["status"])   # insert_user APPLIED ...

What just happened

  • Each insert_user call became a journalled effect, executed live against a SQLite savepoint.
  • rollback() folded the journal backward, restoring each savepoint — so the writes simply never happened.
  • Leaving the block with no rollback folded forward and committed.
